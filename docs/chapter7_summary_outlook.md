7 总结与展望

本章对数字化养殖大棚监控系统的设计与实现工作进行总结，并结合系统当前完成情况分析存在的不足，最后对系统后续智能化发展方向进行展望。本系统围绕养殖大棚环境监测、设备控制、告警管理和养殖档案管理等需求展开，实现了硬件采集、串口桥接、后端处理、前端展示和业务管理的基本闭环，为传统养殖场景的数字化改造提供了一个可运行、可演示、可扩展的系统原型。

7.1 工作总结

本课题以数字化养殖大棚为应用场景，针对传统养殖环境监测依赖人工、设备控制响应不及时、生产档案记录分散和异常情况追溯困难等问题，设计并实现了一套基于物联网技术的养殖大棚监控系统。系统采用分层架构进行设计，将硬件采集端、数据传输层、后端业务层和前端应用层进行划分，使各部分职责清晰，便于开发、调试和后续扩展。

在系统实现过程中，硬件端以 STM32 控制板为核心，完成了温湿度采集、OLED 本地显示、继电器控制和串口数据输出等功能。软件端使用 FastAPI 搭建后端服务，使用 SQLAlchemy 完成数据库模型设计和数据访问，使用 Vue 3 构建前端页面，并通过 ECharts 实现环境数据趋势可视化。同时，项目设计了 serial_bridge.py 串口桥接程序，将 STM32 串口数据转发到后端接口，使硬件端和 Web 系统能够完成联调。

系统围绕实际养殖管理业务划分为环境监测、设备控制、告警管理、养殖档案、生产任务和系统管理等模块。环境监测模块负责接收设备上报的温度、湿度、二氧化碳浓度和氨气浓度等数据，并提供最新数据和历史数据查询；设备控制模块支持人工控制和自动规则控制；告警管理模块能够根据环境阈值生成异常告警；养殖档案模块支持批次档案和个体档案管理。通过这些模块，系统初步实现了对养殖大棚环境和生产信息的集中管理。

7.1.1 系统成果总结

本系统的主要成果体现在以下几个方面。

第一，完成了养殖大棚环境数据采集与上报链路。硬件端能够采集温湿度数据，并通过串口输出 JSON 格式的 telemetry 数据。桥接程序能够读取 COM5 串口数据，并调用后端 /api/v1/telemetry/ 接口上传环境数据。后端接收数据后能够完成设备自动注册、环境数据入库和最新数据缓存，前端能够显示设备最新环境指标和历史趋势图。该链路是当前系统最稳定的核心功能。

第二，完成了后端业务接口和数据模型设计。系统后端已经实现环境监测、设备管理、设备控制、告警管理、养殖档案、生产运营和系统管理等接口。数据库设计包括用户表、设备表、环境数据表、告警信息表、控制命令日志表、自动化规则表、养殖批次档案表和个体档案表等，能够支撑系统主要业务功能。后端采用服务层封装业务逻辑，使接口层和业务处理保持相对分离。

第三，完成了前端监控和管理页面实现。前端系统基于 Vue 3 开发，包含首页概览、环境监测、设备控制、告警管理、养殖档案、生产任务和系统管理等页面。环境监测页面能够按棚区和设备展示实时数据，趋势图支持温度、湿度、CO2 和氨气指标切换；设备控制页面能够展示风机、水帘和补光灯等组件；告警页面能够展示待处理告警和风险统计；档案页面能够维护批次和个体养殖信息。

第四，完成了告警判断和自动控制基础逻辑。系统后端根据环境数据进行阈值判断，当温度、湿度、二氧化碳浓度或氨气浓度超出设定范围时生成告警记录。控制服务中预设了高温启动风机、高温联动水帘、氨气超标启动风机和温度恢复关闭风机等自动化规则。通过这些规则，系统具备了从环境监测到异常告警、再到控制命令生成的基础能力。

第五，完成了软硬件联调框架。项目通过 serial_bridge.py 将 STM32 串口与 FastAPI 后端连接起来，实现了上行 telemetry 数据上传和下行控制命令轮询。桥接程序能够将后端 pending 命令转换为硬件端可识别的 JSON 格式，并在收到硬件 ACK 后回写命令状态。虽然控制下行稳定性仍需进一步验证，但系统已经具备完整控制闭环的代码基础。

总体来看，本系统已经完成毕业设计所需的主要功能实现，能够展示物联网养殖大棚监控系统的基本工作流程。系统不仅能够监测环境数据，还能够结合告警、控制和档案管理进行综合展示，体现了数字化养殖管理的基本思路。

7.2 不足与改进

虽然本系统已经实现了主要功能，但由于时间、硬件条件和项目规模限制，系统仍存在一些不足。对这些问题进行分析，有助于明确后续优化方向，也能使系统更接近真实养殖场景下的应用需求。

7.2.1 系统不足

第一，硬件传感器接入仍不完整。当前系统中温湿度采集链路相对稳定，前端能够显示真实温湿度数据；但 CO2 和氨气传感器在当前联调阶段仍以字段预留、模拟值或 null 值为主。由于气体浓度是养殖大棚环境安全的重要指标，后续需要接入真实传感器并进行标定，确保采集数据具有实际参考价值。

第二，控制下发链路稳定性仍需提升。当前前端、后端和桥接程序已经能够生成和发送控制命令，桥接程序也可以将命令写入串口。但实际测试中，前端人工控制板子动作的稳定性仍受 PC 到 STM32 下行串口链路影响。如果只有串口发送日志而没有硬件 ACK，说明命令可能未被硬件稳定接收或解析。因此，后续需要重点验证 COM5 到 STM32 的下行通信，必要时使用外接 USB-TTL 模块进行单独测试。

第三，设备真实状态上报不够完善。目前系统控制页面中的部分设备状态依赖命令日志或规则推断，硬件端尚未完整上报继电器真实状态。实际应用中，仅凭命令记录不能完全代表设备真实运行状态，例如继电器故障、线路断开或执行设备异常时，系统可能无法及时发现。因此，后续应在 telemetry 数据中增加风机、水帘、补光灯和继电器状态字段，实现控制结果的真实反馈。

第四，自动化规则仍偏演示性质。当前系统已内置高温启动风机、氨气超标启动风机等规则，但规则设计仍较简单，主要基于单一阈值判断。真实养殖场景中，设备控制需要考虑季节、昼夜、养殖品种、生长阶段、棚内通风条件和设备能耗等因素。后续应结合实际养殖经验，完善规则优先级、回差控制、手自动切换和联动策略。

第五，数据库和部署方案仍需规范化。当前项目使用 SQLite 进行开发联调，适合毕业设计演示和单机测试，但在多设备、长期运行和多人管理场景下，SQLite 的并发能力和维护能力有限。后续可迁移到 MySQL 或 PostgreSQL，并引入正式数据库迁移工具。同时，当前数据库初始化主要依赖 create_all 和少量运行时补字段逻辑，后续应建立规范的迁移和版本管理流程。

第六，系统安全和权限控制仍不完善。系统中已经设计了用户表和角色字段，但当前接口权限控制和登录认证功能尚不完整。在实际部署中，应增加用户登录、Token 鉴权、角色权限控制、操作审计和敏感接口保护，防止未授权用户访问设备控制、档案修改和系统管理等功能。

第七，性能测试和长期稳定性验证仍较有限。当前测试主要基于本地单设备联调，能够验证主流程可用，但尚未进行多设备并发上报、长时间运行、异常网络、数据库大量数据和高频控制命令等复杂场景测试。后续需要设计更完整的测试方案，以验证系统在真实养殖场景中的稳定性和可靠性。

7.3 未来展望

随着物联网、人工智能和大数据技术的发展，养殖大棚管理正在从人工经验驱动逐步转向数据驱动和智能决策。本系统已经实现了数字化养殖大棚监控的基础能力，后续可以在智能感知、智能控制、数据分析和平台化管理等方向继续扩展。

7.3.1 智能化发展方向

第一，完善多源传感器接入，实现更全面的环境感知。后续系统可以接入更多类型的传感器，如光照强度、风速、噪声、PM2.5、水质、饲料余量和动物活动传感器等。通过多源数据采集，系统能够更全面地反映养殖环境和动物状态，为精细化管理提供数据基础。

第二，引入更智能的控制算法。当前系统主要采用固定阈值规则进行告警和控制，后续可以引入模糊控制、PID 控制或基于历史数据的预测控制算法。例如，系统可以根据温度变化趋势提前启动通风或降温设备，而不是等到温度超过阈值后再响应。这样能够减少环境波动，提高动物舒适度，并降低设备能耗。

第三，构建养殖数据分析与预测模型。随着环境数据、告警数据、设备运行数据和养殖档案数据的积累，系统可以进一步分析环境变化与动物生长、健康状态、饲料消耗之间的关系。通过数据建模，可以实现异常趋势预警、生长状态评估、出栏时间预测和疫病风险提示等功能。

第四，提升设备状态感知和运维能力。后续可以增加继电器真实状态、电流检测、设备运行时长和故障反馈等数据采集，实现设备健康管理。当风机长时间运行、设备状态异常或维护周期到达时，系统可以自动生成维护提醒，提高设备管理效率。

第五，扩展多棚区和云端集中管理能力。当前系统主要面向本地单机联调，后续可扩展为多棚区、多设备和多用户管理模式。通过 MQTT、NB-IoT、4G/5G 或云平台接入，可以实现远距离设备连接和集中监控，使管理人员能够通过 Web 或移动端查看多个养殖区域的运行状态。

第六，加强系统安全和权限体系。未来系统应完善登录认证、角色权限、接口鉴权、操作日志和数据备份机制。对于设备控制、告警处理、档案修改等关键操作，应记录操作人、操作时间和操作内容，保证系统具有可追溯性和安全性。

第七，优化前端交互和移动端适配。养殖管理人员在实际工作中可能更多使用手机或平板查看数据，因此后续可以增加移动端适配，提供更简洁的移动监控页面、告警推送和远程控制功能。同时，可以优化前端图表、风险面板和设备地图，使系统更直观易用。

综上所述，本系统完成了数字化养殖大棚监控系统的基础设计与实现，验证了物联网技术在养殖环境监测和设备控制中的可行性。虽然系统仍存在真实传感器接入、控制链路稳定性、权限管理和长期运行验证等不足，但整体架构具有较好的扩展性。未来通过进一步完善硬件采集、智能控制、数据分析和云端管理能力，系统可以逐步发展为更智能、更稳定、更适合实际生产应用的智慧养殖管理平台。
