# 项目阶段总结

更新时间：2026-04-20

## 当前定位

本项目是一个联网数字化养殖大棚监控系统，目前处于“主链路联调已跑通，功能仍需完善”的阶段。

不能再按旧文档中的“生产就绪”或“100% 完成”理解当前代码。更准确的判断是：

- 环境数据上报链路已经可用
- 前端可以展示真实 telemetry 数据
- 后端基础 API、入库、告警、控制命令框架已经具备
- STM32 本地采集、OLED 显示、继电器基础逻辑已经可用
- 人工控制板子动作仍需重点验证和完善

---

## 已经完成的主要能力

### 后端

- FastAPI 服务可启动
- SQLite 数据库可用
- telemetry 数据可入库
- 设备可自动注册
- 告警逻辑可执行
- 自动控制规则可执行
- 待执行命令接口可用
- 命令状态回写接口可用
- 数据库 Session 释放问题已修复

### 前端

- Vue 前端可启动
- 能显示环境监测数据
- 支持监测、控制、档案、告警、任务、系统等页面
- 当前页面时钟 1 秒刷新
- 当前模块数据 5 秒静默刷新

### 桥接

- `tools/serial_bridge.py` 可读取 STM32 串口数据
- 可将 telemetry 上报到 FastAPI
- 可轮询后端 pending 命令
- 可向串口写入紧凑 JSON 命令
- 可处理 STM32 ACK 并回写后端

### 硬件

- `hardware/demo` 是当前主工程
- OLED 显示可用
- SHT3X 温湿度采集可用
- 继电器基础控制可用
- 温度阈值与回差控制已实现
- telemetry JSON 上报已实现
- 命令解析和 ACK 代码已写入固件

---

## 当前未完成或不稳定的部分

### 人工控制板子动作不稳定

这是当前最大缺口。

目前前端、后端和桥接层都能产生并发送命令，但板子是否稳定执行命令仍受串口下行链路影响。需要继续验证 `COM5 -> STM32` 是否可靠。

### 设备真实状态未完整上报

当前 telemetry 主要包含：

- 温度
- 湿度
- CO2：`null`
- 氨气：`null`

还未包含：

- 继电器 1 真实状态
- 继电器 2 真实状态
- 风机真实反馈
- 水帘真实反馈

### 自动规则仍需业务化

当前规则更偏联调演示。后续需要明确：

- 高温时是否只开风机，还是联动水帘
- 手动控制是否覆盖自动控制
- 手动控制后何时恢复自动
- `fill_light` 是否保留在当前硬件演示中

### 数据库迁移还不正式

当前使用 `create_all` 和少量运行时补字段逻辑。后续如果进入稳定版本，应引入正式迁移方案。

### 文档仍需持续同步

部分旧文档曾经写得过满，已经开始修正，但后续新增功能后仍需要及时更新。

## 当前推荐运行方式

推荐用 VS Code 任务：

```text
Tasks: Run Task -> app: dev
```

它会启动：

- 后端
- 前端
- 桥接脚本

手动启动方式见 `docs/INTEGRATION_GUIDE.md`。
